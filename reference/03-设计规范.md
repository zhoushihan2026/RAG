#  UI 设计规范

> 本文档针对 AI 生成前端代码中常见的 5 类设计问题，给出具体的审查标准和修改指令。后续所有前端页面开发前，必须先对照本规范自检。

---

## 1. 卡片嵌套过多

### 标准

只有当一组内容**真的需要独立成块、可点击、可复用或需要对比**时，才使用卡片（Card）。普通的信息分组，优先使用留白、标题层级、分割线和背景色差来处理。

### 禁止项

- 卡片内部又嵌套了卡片；
- 内容被包裹在卡片中，但没有明确的理由；
- 重复出现的阴影、边框和圆角容器。

### 修改方式

- 移除不必要的卡片包裹层；
- 使用间距、标题、分割线和对齐方式来对内容进行分组；
- 仅在内容需要视觉上独立或可点击时才保留卡片。

---

## 2. 紫蓝渐变、玻璃拟态、发光按钮

### 标准

不要用"科技感"来替代品牌判断。先确定产品的气质，再选择颜色。渐变和发光（glow）效果只能作为**少量的强调色（accent）**，不要同时出现在背景、标题、按钮、卡片边框和图标上。

### 禁止项

- 滥用紫/蓝渐变；
- 霓虹发光效果；
- 把玻璃拟态（glassmorphism）当作纯装饰；
- 在重要标题上使用渐变文字。

### 替代方案

- 定义一套克制的品牌调色板；
- 仅使用一种主强调色；
- 文本尽量保持纯色；
- 仅在需要表达交互或空间深度时才使用发光效果；
- 确保所有文本与背景的组合都有足够的对比度。

### 面经助手色彩方案

| 类型 | 修改前（AI 默认） | 修改后（克制设计） |
|------|------------------|-------------------|
| 主强调色 | 紫蓝渐变 #7C3AED → #3B82F6 | 纯色靛青 #2563EB |
| 背景色 | 深色渐变 + 玻璃拟态 | 纯白 #FFFFFF / 浅灰 #F8FAFC |
| 标题文字 | 渐变紫蓝 | 深灰 #1E293B |
| 正文文字 | 浅灰/半透明 | 中灰 #475569 |
| 卡片边框 | 发光 + 半透明 | 细线 #E2E8F0 或无 |
| 按钮 | 发光渐变 | 纯色 + hover 状态 |

## 3. 没有信息层级和重点

### 标准

先建立字体排版比例（typography scale）。H1、H2、正文、说明文字（caption）、标签（label）和按钮文本之间，必须在大小、粗细和间距上有明确的差异。不要仅仅依靠颜色来区分信息层级。

### 审查项

- H1、H2、正文、标签、说明文字和按钮在视觉上是否有明显区分；
- 字体大小是否过于接近；
- 多行文本的行高是否舒适；
- 是否有正文过小或字间距过紧的情况。

### 排版比例系统

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| H1 | 32px | 700 | 1.2 | 页面主标题 |
| H2 | 24px | 600 | 1.3 | 区块标题 |
| H3 | 18px | 600 | 1.4 | 卡片/模块标题 |
| 正文 | 14px | 400 | 1.6 | 描述、内容 |
| 说明文字 | 12px | 400 | 1.5 | 辅助信息、时间戳 |
| 标签 | 11px | 500 | 1.4 | 分类标签、状态标识 |
| 按钮 | 14px | 500 | 1 | 按钮文本 |

### 修改方式

- 建立更清晰的字体比例系统；
- 增加标题与正文之间的视觉对比度；
- 正文的行高设置在 **1.5 到 1.7** 之间；
- 避免对长文本使用全大写；
- 保持标签较小，但必须清晰可读。

## 4. 过度居中排布

### 标准

Hero 区域（首屏）可以居中，但正文、功能说明、产品解释、Dashboard 内容应尽量左对齐。页面需要有清晰的阅读动线，而不是把所有模块都在正中间堆叠。

### 禁止项

- 标题、正文、卡片和按钮默认全部居中；
- 居中对齐导致内容难以快速扫读；
- 页面看起来像是重复的落地页（Landing Page）模板。

### 修改方式

- 仅在 Hero 区域或最终的 CTA（行动号召）区块保留居中对齐；
- 解释性内容使用左对齐；
- 在适当的地方引入不对称布局；
- 建立从"标题"到"细节"再到"行动"的清晰阅读动线。

### 面经助手各页面对齐规则

| 页面/区域 | 对齐方式 | 说明 |
|----------|----------|------|
| 首屏 Hero | 居中 | 品牌展示，允许居中 |
| Dashboard 数据区 | 左对齐 | 快速扫读面试数据 |
| 面经列表 | 左对齐 | 标题左对齐，操作右对齐 |
| 题目详情 | 左对齐 | 阅读动线：题目 → 答案 → 操作 |
| 练习模式 | 居中/左混合 | 题目居中，选项/答案左对齐 |
| 设置/表单 | 左对齐 | 标签左对齐，输入框跟随 |

---

## 5. 模板化图标和容器

### 标准

把"上方居中的圆角图标框"改成"标题左侧的小图标"，去掉图标的外层容器，并把图标尺寸控制在接近正文行高。让图标起到辅助识别的作用，而不是成为卡片的视觉中心。

### 禁止项

- 圆角方形图标平铺在标题正上方；
- 图标尺寸过大，喧宾夺主；
- 所有卡片都采用完全相同的"图标-标题-描述"布局。

### 修改方式

- 移除圆角方形的图标容器；
- 将图标移至标题左侧，改为行内排列；
- 缩小图标尺寸，使其与标题高度对齐；
- 保持标题作为最强的视觉元素；
- 避免所有卡片都采用完全相同的"图标-标题-描述"布局。

### 图标使用规范

| 场景 | 图标位置 | 图标尺寸 | 容器 |
|------|----------|----------|------|
| 页面标题 | 标题左侧 | 24px | 无容器 |
| 功能模块标题 | 标题左侧 | 18px | 无容器 |
| 列表项 | 标题左侧 | 16px | 无容器 |
| 按钮内 | 文本左侧 | 14px | 无容器 |
| 状态标识 | 文本右侧/独立 | 12px | 无容器 |

## 自检清单

每次生成或修改前端页面后，对照以下清单检查：

- [ ] 没有卡片嵌套卡片的情况
- [ ] 没有使用紫/蓝渐变、发光效果、玻璃拟态
- [ ] 标题、正文、标签的字号差异明显
- [ ] 正文行高在 1.5~1.7 之间
- [ ] Dashboard/列表/详情页内容左对齐
- [ ] 图标在标题左侧，无圆角容器
- [ ] 所有文本与背景对比度足够
